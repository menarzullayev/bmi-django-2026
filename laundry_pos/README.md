Дипломная работа: программное обеспечение для учёта заказов и клиентов в химчистке.

## Описание

Web-приложение POS-системы (Point of Sale) для химчистки. Позволяет:

- принимать вещи от клиентов с автоматической генерацией квитанций и QR-кодов;
- вести базу клиентов;
- управлять прайс-листом (матрица «вид одежды × услуга»);
- отслеживать статусы заказов (Принято → Сортировка → Стирка → Сушка → Глажка → Готово → Выдано);
- принимать оплаты (наличные / карта / перевод);
- планировать маршруты доставки и отмечать выполнение;
- формировать отчёт за день (по статусам, методам оплаты, кассирам);
- печатать чек (PDF, ReportLab, шрифт DejaVuSans с кириллицей);
- предоставлять клиенту мобильную страницу статуса по QR-коду без авторизации.

## Технологии

- Python 3.12, Django 5.1
- SQLite
- Tailwind CSS (CDN), HTMX, Lucide Icons (CDN), шрифт Nunito
- Pillow, qrcode (генерация QR), ReportLab (PDF-чеки)

## Структура проекта

```
config/          — настройки Django, корневые URL
accounts/        — пользователи (роли: admin, cashier, operator, driver)
customers/       — клиенты химчистки
catalog/         — виды одежды, услуги, прайс-лист
intake/          — приём заказов, квитанции, вещи, печать
payments/        — оплаты
delivery/        — маршруты доставки и задания
reports/         — отчёт за день

```

## Карта URL

- `/login/` — вход
- `/pos/` — главная панель (KPI)
- `/intake/new/` — новый приём
- `/tickets/` — список квитанций (фильтр по статусу, телефону, дате)
- `/tickets/<id>/` — детали квитанции
- `/tickets/<id>/receipt/` — HTML-чек
- `/tickets/<id>/receipt/pdf/` — PDF-чек 80мм
- `/pricing/` — матрица цен (inline edit через HTMX)
- `/catalog/cloth-types/`, `/catalog/service-types/` — справочники
- `/delivery/` — маршруты доставки
- `/delivery/route/new/` — создание маршрута
- `/payments/` — история оплат
- `/reports/daily/?date=YYYY-MM-DD` — отчёт за день
- `/qr/<номер-квитанции>/` — публичная страница статуса (без авторизации)
- `/admin/` — Django-админка

## Чек и QR

- Каждой новой квитанции автоматически присваивается номер `T-YYYYMMDD-XXXX` и генерируется PNG QR-код, который указывает на `/qr/<номер>/`.
- PDF-чек печатается на узкой полосе (80 мм) с кириллической поддержкой через DejaVuSans (если найден в системе), иначе fallback на Helvetica.
